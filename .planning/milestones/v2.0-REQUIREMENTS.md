# Requirements: Tokenizor v2.0 — Deep Understanding

**Defined:** 2026-03-12
**Core Value:** Reduce agent round-trips by providing complete compilable context in single calls, enable navigation of type hierarchies and trait relationships, and surface temporal/structural metadata that guides agent decision-making.

**Origin:** Cross-model analysis (Gemini weak + strong models, validated by Claude against codebase). Items prioritized by effort/value ratio against actual Tokenizor architecture.

## Recursive Type Resolution ✅ (shipped v0.6.0, 2026-03-12)

- [x] **TYPR-01**: `get_context_bundle` returns transitive type definitions used in the focus symbol's signature and body
- [~] **TYPR-02**: Depth limit hardcoded at 2 levels (not yet configurable via parameter) — prevents unbounded recursion on deeply nested types
- [x] **TYPR-03**: Primitives and std library types excluded from resolution (String, i32, Vec, Option, Result, etc.)
- [x] **TYPR-04**: Resolved types returned as a `dependencies` list (name, kind, file, lines, body) alongside existing callers/callees
- [x] **TYPR-05**: Resolution works across files — if `UserConfig` is defined in `config.rs` and used in `handler.rs`, the bundle includes it

## Trait/Interface Implementation Mapping ✅ (shipping v0.8.0, 2026-03-12)

- [x] **IMPL-01**: New `ReferenceKind::Implements` variant in xref system
- [x] **IMPL-02**: Tree-sitter extraction for 9 languages: Rust, TypeScript, Java, C#, Python, C++, Ruby, PHP, Swift (Go skipped — structural typing)
- [x] **IMPL-03**: New `find_implementations` tool — given a trait/interface name, returns all implementing types with file locations
- [x] **IMPL-04**: Reverse direction: given a struct, show which traits it implements (via `direction: "type"` parameter)
- [x] **IMPL-05**: Results integrate with existing `OutputLimits` for bounded output

## Enriched File Context ✅ (shipped v0.7.0, 2026-03-12)

- [x] **ECTX-01**: `get_file_context` enriched view shows explicit import/export summary ("Imports from N sources", "Used by M files")
- [x] **ECTX-02**: Import sources grouped by qualified name with symbol counts
- [x] **ECTX-03**: Export consumers grouped by file with reference counts
- [x] **ECTX-04**: Data sourced from existing reverse index and xref tables — no new indexing required

## Dependency Visualization ✅ (shipped v0.5.0, 2026-03-12)

- [x] **DVIZ-01**: `find_dependents` supports optional `format` parameter: `text` (default), `mermaid`, `dot`
- [x] **DVIZ-02**: Mermaid output produces valid flowchart with module-level nodes and directional edges
- [x] **DVIZ-03**: DOT output produces valid Graphviz digraph
- [x] **DVIZ-04**: Output respects existing `limit`/`max_per_file` bounds

## Git Temporal Context

- [ ] **GTMP-01**: On startup, parse recent git history (configurable depth, default 50 commits) via `git2` crate or `git log`
- [ ] **GTMP-02**: Per-file churn score (modifications per time window) stored in index metadata
- [ ] **GTMP-03**: Co-change matrix — files frequently modified together flagged as coupled
- [ ] **GTMP-04**: `get_file_context` includes `last_modified`, `churn_frequency`, and `often_changed_with` when git data available
- [ ] **GTMP-05**: Graceful degradation — if not a git repo or git unavailable, features simply absent from output

## Constraints

- **Binary size**: No changes that increase release binary by >5MB (rules out shipping embedding models)
- **Startup time**: Git history parsing must not add >500ms to startup for repos with <1000 commits
- **Zero new runtime dependencies**: No external services required (local-first principle preserved)
- **Backward compatibility**: All existing tool signatures preserved; new features are additive parameters
